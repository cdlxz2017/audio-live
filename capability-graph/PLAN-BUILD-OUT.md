## 搭建方案：分三阶段

### 阶段一：建骨架（今天可做）

**目标：让 SOP 能跑起来，不是从零开始**

| 任务 | 内容 | 耗时 |
|------|------|------|
| 1. 创建目录结构 | tools/ systems/ risk-patterns/ pitfalls/ frameworks/ | 5分钟 |
| 2. 建立卡片模板 | 每种卡片的 Markdown 模板 | 10分钟 |
| 3. 填充已有知识 | 把现有的技术知识迁移为卡片格式 | 30分钟 |

**已可迁移的知识来源**：
- `TOOLS.md` → tools/ 卡片
- `MEMORY.md` 中的系统配置 → systems/ 卡片
- 历史踩坑记录 → pitfalls/ 卡片
- `memory/TASK-SOP.md` → frameworks/ 卡片

### 阶段二：在实践中积累（1-2 周）

**目标：每次任务都沉淀，让卡片库越来越丰富**

| 任务 | 说明 |
|------|------|
| 每次 Standard/Complex 任务后 | 按 SOP Step 9 更新对应卡片 |
| 每次失败后 | 写入 risk-patterns/ |
| 每次发现新坑 | 写入 pitfalls/ |
| 每周回顾 | 清理重复卡片，合并相关卡片 |

### 阶段三：自动化与优化（1 个月后）

**目标：减少手动维护，让知识沉淀更自动**

| 任务 | 说明 |
|------|------|
| 自动检测新工具 | 发现新命令/工具 → 自动创建卡片草稿 |
| 失败模式自动归类 | 失败日志 → 自动写入 risk-patterns/ 草稿 |
| 卡片质量检查 | 定期检查卡片完整性，标记过时内容 |
| 分级规则调优 | 根据实际执行情况调整评分阈值 |

## 卡片模板设计

```markdown

## 基本信息
- **类型**：CLI / API / Skill / 内置命令
- **路径/URL**：
- **状态**：✅ 正常 / ⚠️ 有限制 / ❌ 不可用

## 核心能力
- 能力 1
- 能力 2

## 适用场景
- 场景 1
- 场景 2

## 已知限制
- 限制 1
- 限制 2

\`\`\`

\`\`\`

- 坑 2：描述 + 解决方案

| 日期 | 任务 | 结果 |
|------|------|------|
```
