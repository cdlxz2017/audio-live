### 二、关键技术问题及解决

#### 1. Fastify v5 + @fastify/cors v9 版本冲突
**现象**：服务启动后立即崩溃，日志无输出
**解决**：全量降级 `npm install fastify@4 @fastify/cors@8`

#### 2. pino-pretty 日志配置错误
**现象**：logger transport target 不存在
**解决**：统一设置 `logger: false`，移除所有 transport 配置

#### 3. TypeScript 编译错误（100+ → 0）
**主要类型**：
- `Exception` → `Error` 后缀统一
- `event_id_service_name_payload` → `event_id_service_name`（Prisma 复合主键字段名变更）
- Zod schema 直接传给 Fastify（Fastify v4 需要 JSON Schema）
- `HeavenlyEvent` 类型缺少 `id`/`version`/`timestamp` 字段
- `AppError` 被错误设为 `abstract`
- `RedisEventBus` 构造函数参数不匹配（需传入 `Redis` 实例 + `serviceName`）

#### 4. @tiandao/common-event 发布订阅修复
**问题**：`publish(event: HeavenlyEvent)` overload 和 `subscribe` 签名不匹配
**解决**：重写 `RedisEventBus` 类，统一参数类型，添加 `EventSubscriber` 别名

#### 5. 公共库编译问题
- `common-exception`：`AppError` 移除 `abstract`
- `common-utils`：添加 `logger` 导出
- `common-event`：重建 dist（因源码修复后未重新编译）
