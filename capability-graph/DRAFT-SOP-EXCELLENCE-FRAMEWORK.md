## 一、核心流程图

```
收到问题/任务
    │
    ▼
【Step 0】问题识别
    │
    ├─ 提取关键词/意图/领域
    │
    ▼
【Step 1】历史检索 —— 之前解决过吗？
    │
    ├─ 是 → 调取历史成功案例 → 跳到 Step 4
    │       │
    │       ├─ 确认方案是否仍适用（环境/版本有无变化）
    │       │   ├─ 是 → 直接复用
    │       │   └─ 否 → 进入 Step 2
    │       │
    │       └─ 检查历史方案的成功率
    │           ├─ 一次成功 → 直接复用
    │           └─ 多次尝试 → 注意历史踩坑点
    │
    └─ 否 → 进入 Step 2（新方案设计）
```

## 二、已知问题处理流程

### Step 1：历史方案检索

**查询路径：**
1. 能力图谱（capability-graph/）— 工具使用记录
2. 副脑 Problem Thread — 是否有 in_progress/completed 的 Thread
3. 主脑记忆召回 — 语义搜索相关历史对话
4. 技术文档库（tech-knowledge）— 是否有相关技术文档

**匹配规则：**
- 关键词匹配（标题/领域标签）
- 语义相似度 > 0.7
- 同一 domain 下的相关问题

### Step 2：方案确认（微调）

- 读取历史方案的完整记录（Problem → Analysis → Decision → Implementation → Verification）
- 检查环境变量变化（版本/配置/依赖是否还一样）
- 如有变化 → 进入 Step 3 重新设计
- 如无变化 → 跳到 Step 4

### Step 3：方案重新设计（仅当历史方案不再适用）

见下方"新方案设计流程"。

### Step 4：危险点识别 ⚠️

**读取历史方案中的危险点记录**（如有），然后补充：

| 检查维度 | 问题 |
|---------|------|
| 数据安全 | 会不会丢失/损坏用户数据？ |
| 系统稳定 | 会不会导致服务宕机/中断？ |
| 依赖影响 | 会不会影响其他模块/系统？ |
| 回退能力 | 出错了能回滚吗？怎么回滚？ |
| 权限风险 | 是否需要 elevated 权限？是否安全？ |
| 外部影响 | 会不会对外部系统/服务产生影响？ |

### Step 5：工具手册查阅

- 读取对应工具的"工具卡片"（capability-graph/tools/）
- 确认能力边界、已知限制、推荐用法
- 如工具卡片不存在 → 临时查阅官方文档并提炼要点

### Step 6：执行

- 按方案步骤执行
- 每完成一步，验证结果
- 遇到危险点 → 按预规避措施处理
- 遇到意外 → 暂停 → 报告主人

### Step 7：文档更新

- 更新对应的 Thread（副脑）
- 更新对应的工具卡片/系统卡片（能力图谱）
- 如有新坑 → 写入「避坑指南」
- 更新当日记忆文件（memory/YYYY-MM-DD.md）

## 三、新问题处理流程

### Step 1：问题定义

- 明确问题是什么
- 明确目标是什么
- 明确约束条件（时间/资源/风险容忍度）

### Step 2：多方案设计

**至少设计 2-3 个方案：**

| 方案 | 描述 | 优点 | 缺点 | 风险等级 | 预估时间 |
|------|------|------|------|---------|---------|
| A | | | | 低/中/高 | |
| B | | | | 低/中/高 | |
| C | | | | 低/中/高 | |

**选择标准：**
1. 能最好地解决问题
2. 对系统影响最小
3. 回退成本最低
4. 执行速度最快（前三条都满足时）

### Step 3：方案确认

- 向主人呈报方案对比，等待选择
- 或按上述标准自行选择（仅当主人授权）

### Step 4：危险点识别 ⚠️

对选中的方案进行完整的危险点扫描：

```
方案 → 拆解为步骤
每个步骤 → 识别可能的危险点
每个危险点 → 制定规避措施
```

**危险点模板：**
```markdown
### 危险点 #N
- **位置**：哪个步骤
- **风险**：会发生什么
- **影响**：严重程度（高/中/低）
- **规避**：怎么预防
- **应急**：出了事怎么救
```

### Step 5：技术手册查阅

- 查阅涉及系统的技术文档
- 查阅涉及工具的能力卡片
- 理解原理，不做"黑箱操作"

### Step 6：执行

同已知问题流程。

### Step 7：文档更新 + 建库

**新问题特有的步骤：建库**

- 在副脑创建新 Thread（记录完整生命周期）
- 在能力图谱新建/更新相关卡片
- 写入避坑指南
- 标记为"可复用案例"
