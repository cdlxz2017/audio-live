## 总体设计思路（来自 Problem Thread 设计方案）

**核心原则：**

1. 每个 session = 我的一天，需要记录和传承
2. 先问「要什么记忆」，再问「怎么存储」
3. 存的是「过程」，取的是「上下文」
4. 以「问题 / 事件」为主线，而非时间或类型
5. 多线程并行，一个 session 有 N 个事情同时跑

**数据库选型：PostgreSQL + Neo4j 混合**

- PostgreSQL：Thread 核心数据（5 个 Stage 结构化内容）+ pgvector 向量检索
- Neo4j：Thread ↔ Thread 关系、Thread ↔ Session 双向链表
- 与现有架构一致，迁移时两库分别 dump/restore

2. **Neo4j：创建 Thread 节点类型**
   - 标签：`:Thread`

3. **Session 结束时提取逻辑**

4. **Session 启动时加载逻辑**
