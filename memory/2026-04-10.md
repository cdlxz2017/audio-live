## graphify-opus-manager 问题修复与健康检查整合（04:45-05:08）

### 修复的问题

**1. query-layer.js PostgreSQL prepared statement 错误**
- 错误：`pg.Client.query()` 不支持第三参数传 config 对象（`{ name: '...' }`）
- 原因：pg 库中 `Client.query()` 的签名是 `query(text, params, callback)`，config 对象被当成 callback 处理
- 修复：移除 `{ name: 'graphify-memory-exact' }` 等参数，改用匿名查询避免缓存冲突
- 影响：query API (`POST /query`) 持续报错，但三曾仍然 running（降级运行）

**2. health-check.js 日志解析 bug**
- 错误：PM2 日志格式 `150|graphify | { "overall": "degraded"...` 解析失败
- 原因：JSON 跨行时braceCount 计算逻辑有误，找不到完整 JSON 块
- 修复：改用正则 `"overall":` 关键字向上最近一个 `{` 作为起点，收集足够行数后正则提取
- 同时添加 HTTP API fallback（query 层端口 31234 响应即认为 running）

**3. NOGROUP 错误（历史遗留，已自愈）**
- Redis stream `graphify:events` 存在，consumer group `graphify-bridge-cg` 已创建
- 进程重启后 NOGROUP 错误已清除
