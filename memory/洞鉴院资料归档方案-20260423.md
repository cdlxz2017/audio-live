## 三、实施方案：新建 search_history 表

### 3.1 建表 SQL

```sql
CREATE TABLE search_history (
  id            SERIAL PRIMARY KEY,
  query         VARCHAR(500) NOT NULL,           -- 搜索关键词
  results       JSONB NOT NULL,                  -- 完整搜索结果（JSON数组）
  result_count INTEGER NOT NULL,                 -- 结果数量
  user_id       INTEGER,                         -- 用户ID（未登录为NULL）
  ip_address    VARCHAR(45),                     -- 请求IP
  user_agent    VARCHAR(500),                    -- 浏览器UA
  archived_at   TIMESTAMP DEFAULT NOW()          -- 归档时间
);

-- 按关键词+日期去重索引
CREATE INDEX idx_search_query_date ON search_history (query, DATE(archived_at));
-- 按归档时间倒序索引（利于查询最近记录）
CREATE INDEX idx_search_archived_at ON search_history (archived_at DESC);
```

### 3.2 保留策略

- 保留 **90天**
- 定期清理 cron：`DELETE FROM search_history WHERE archived_at < NOW() - INTERVAL '90 days'`

### 3.3 路由改动

**文件**：`/home/ai/projects/tiandao_dongjianyuan/backend/src/routes/search.js`

**改动**：在 `res.json(result)` 之后，加一行 fire-and-forget 异步写入

```js
// 改后
res.json(result);
// 异步归档，不等待
query(
  `INSERT INTO search_history (query, results, result_count, user_id, ip_address, user_agent, archived_at)
   VALUES ($1, $2, $3, $4, $5, $6, NOW())`,
  [q, result.results, result.results.length, req.user?.userId, req.ip, req.get('User-Agent')]
).catch(err => console.error('[Search] 归档失败:', err.message));
```

### 3.4 数据库连接
