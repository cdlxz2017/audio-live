### 3.2 Gemini TTS（Google 插件）
最新版本在 Google 插件中内置了 Gemini TTS（text-to-speech）。

**我们现状**：目前使用有道 TTS。

**行动**：评估 Gemini TTS 质量（无需 API key？或用已有 Google API key）。

---

### 3.3 LanceDB 云端记忆（Cloud-backed LanceDB memory）
最新版本支持 LanceDB 作为记忆后端（云端备份版）。

**我们现状**：使用 PostgreSQL + pgvector 的三层记忆系统。

**评估结论**：LanceDB 是嵌入式向量数据库，适合大规模向量检索，但我们的主脑 PostgreSQL 已稳定，暂不迁移。

---

### 3.4 Copilot Embeddings（嵌入模型）
最新版本支持 Copilot embeddings 作为新的嵌入选项。

**我们现状**：使用 BGE-m3（Ollama 本地）。

**评估结论**：BGE-m3 在本地运行，无 API 费用，已稳定，暂不更换。

---

### 3.5 Model Auth Status Card
新增模型认证状态卡片，显示各 provider API key 是否配置正确。

**行动**：检查 gateway 配置，验证 API key 状态。

---

### 3.6 Plugin Doctor（插件诊断修复）
`openclaw plugins doctor` 命令可以自动修复插件依赖问题。

**现状**：日志中有 `[plugins] memory-recall-plugin failed to load` 错误。

**行动**：运行 `openclaw plugins doctor` 修复。

## 四、已有但未充分使用的功能

### 4.1 Canvas 实时协作（canvas eval / snapshot）
我们只用了 `canvas present`，还有：
- `canvas eval`：在 canvas 上执行 JavaScript 脚本
- `canvas snapshot`：截取 canvas 当前状态为图片

**应用场景**：洞鉴院 Dashboard 实时截图反馈、定时快照存档。

---

### 4.2 Detached Task Executor（插件任务 detached 模式）
Plugin executors 可以拥有独立的任务生命周期和取消机制，不需要触碰核心任务内部。

**应用场景**：长时间运行的归档任务（如 search_history 批量清理）。

---

### 4.3 Cron jobs-state.json（cron 状态分离）
v2026.4.20 起，Cron 执行状态拆分到 `jobs-state.json`，`jobs.json` 保持为 Git 可追踪的定义文件。

**优势**：Cron job 定义可以纳入 Git 版本控制，不会因为状态变化产生 diff。

**我们现状**：Cron job 在数据库里，jobs.json 文件化后更利于备份和迁移。

---

### 4.4 Compaction Lifecycle Events（压缩生命周期事件）
Session compaction（上下文压缩）时发送 `compaction_start` / `compaction_end` 事件通知。

**应用场景**：在 compaction 时自动触发记忆归档写入。

---

### 4.5 thinking.keep="all" on Kimi K2.6
Moonshot Kimi K2.6 支持 `thinking.keep = "all"`，保留完整思考链。

**我们现状**：TOOLS.md 记录 K2.6 评分 7/10，thinking 模式未知。

---

### 4.6 Tiered Model Pricing（分层模型定价）
支持从缓存目录加载分层定价，并内置 Moonshot K2.6/K2.5 成本估算。

**应用场景**：优化 API 费用，精细化路由到不同价位的模型。
